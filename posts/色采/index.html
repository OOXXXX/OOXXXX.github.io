意义

这篇文章的名字叫“色采”，“采”字译为“采集，收集”的意思。为什么想要写这样一篇文章呢，是因为最近包括前段时间我都在研究一些配色方面的东西，这是设计一款产品必须要经历的步骤。

举个栗子，在一款ins的logo被设计出来之前，设计师们往往是这样创作的：

他们先设计出许多黑白色的版本，在初步定稿之后再加以颜色。这种过程保证了 Logo 即使是在单色的情况下也清晰明了，而最终颜色的加持，能增强 Logo 的整体效果。

为了增强你要传达的内容而存在，这就是颜色最本质的意义。

往往一个颜色输入进计算机，是按照Hex编码的标准来的，这是一种16进制的字符。当然，也有比如说RGB（xx,xx,xx,xx)的颜色编码。解释一下，这前面三个格子填的是红，绿，蓝三种颜色的数值，往往后面一个是颜色的透明度（opacity）。

原料

~Homebrew：macOS 平台上的第三方软件包管理工具，我们将会使用它来安装今天要用到的开源工具spicetify-cli

~spicetify-cli：一个跨平台的Spotify客户端定制工具，支持 Windows，MacOS 和 Linux。

安装

首先是Homebrew的安装，在终端输入以下命令：

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

其次我们需要完成spicetify-cli的安装，在终端中执行以下命令，然后等待安装完成即可：

brew install khanhas/tap/spicetify-cli

基础配置

待安装完成后，依次在终端中执行以下两条命令：

spicetify backup apply enable-devtool

spicetify update

其中第一条命令的意思是进行spicefy备份 第二条命令则是用现有CSS样式进行覆盖

看到success提示后，打开 Spotify 客户端，就能看到你从未体验过的全新 Spotify 了。

:::tip Notice 在Homebrew安装过程中，我曾发生过error报错，查了一下发现竟然是本地网络代理出了问题，所以建议大家尽量用VPN的虚拟IP进行安装。 :::
